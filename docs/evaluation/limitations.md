# Evaluation Limitations

---

## 1. Synthetic Golden Dataset

The eval dataset was generated by `scripts/generate_eval_dataset.py` using controlled synthetic sessions. Real production failure traces may:
- Have more ambiguous failure signals
- Mix multiple failure types in a single session
- Have incomplete trace data (missing `expected_doc_ids`, no `hallucination_flag`)
- Include failure summaries that contradict the actual trace evidence

**Implication:** 100% accuracy on a synthetic dataset does not guarantee the same performance on real traces. The classifier has been tested on real Langfuse traces via the demo agent, but a representative real-world benchmark has not yet been built.

---

## 2. Trace-Only Architecture

Aethen never accesses the monitored agent's KB. This means:

- **Hallucination detection is surface-level** — compares LLM response against retrieved `doc_content` in the trace. If the LLM used training knowledge to give a correct answer but the retrieved docs were wrong, Aethen may misclassify as hallucination.
- **Memory vs blind spot distinction depends on doc_content** — if `doc_content` is missing from the trace, classification falls back to scoring only (less precise).
- **Confidence scores are upper-bounded by trace signal quality** — incomplete traces produce lower confidence scores regardless of actual failure severity.

---

## 3. LLM Judge Subjectivity

The LLM judge (Claude Sonnet 4.6) introduces model-specific bias. A different judge model may rate the same root causes differently. The 85.56% score reflects Claude's evaluation, not an objective ground truth.

The 0–3 scale with normalisation compresses variance — a root cause rated 2/3 by one evaluator might be 3/3 by another.

---

## 4. Confidence Threshold Calibration

Signal weights in `compute_confidence()` are set as universal heuristics (e.g., `doc_id_full_miss = 0.58`). These were calibrated on the synthetic dataset without domain-specific knowledge. For production use with a specific agent type, weights may benefit from fine-tuning based on real failure data.

---

## 5. Context Recall Metric

Context recall measures whether the agent's expected documents were retrieved in the agent's trace — **not** whether Aethen's evidence retrieval (pgvector) found relevant sessions. For memory failure sessions, expected ≠ actual doc IDs by design, so recall is expected to be low. This metric is informational, not a regression gate.

---

## 6. Regression Gate Coverage

Current gates test classification accuracy, keyword match rate, and judge score. They do not test:
- Retrieval latency (pgvector query time)
- End-to-end pipeline latency (9–12 s target)
- Concurrent request handling
- PII redaction effectiveness

These are monitored via Sentry (error rate) and manual smoke tests after deployment.
